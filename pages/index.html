ESP PineTime Bridge

A three-component system for managing daily reminders on a PineTime smartwatch.

What is this?

This project is a full service stack to allow a parent or caretaker to administrate "reminders" on the Open Source InfiniTime smart watch (also known as "PineTime"). The PineTime smart watch is an open source digital watch.

This project makes the tasks of having to flash the firmware, and program the reminders, easy on those that operate them. The technical details of this process are mostly taken care of, and it should be easy enough for this to be deployed by a person with modest computing skills.

This project also aims to be affordable. The PineTime watch itself is inexpensive, and the cost of an ESP32 SoC device similarly is cheap enough that it doesn't cost much more than that and maybe a few cables to get started.

Why an ESP bridge?

One of the more daunting aspects of working with custom watch firmware on the PineTime is that one has to flash the firmware. By deploying the firmware updates on a custom, low cost device that has excellent software support for flashing the watch, we can eliminate the need for third party software, and, the watch firmware can trivially be deployed from the same interface where reminders are managed. This reduces the time needed by caretakers, we get better logging and debugging in return, and the whole experience becomes a lot easier.

The second reason for a dedicated bridge is that we need a relatively capable system to be able to flash the watch, and synchronize reminders to the PineTime. But smart devices may not be appropriate to have in a bedroom of a target user, and the complications of using mobile devices being used for this task are many: Devices may run out of battery, be out of range, have software update and compatibility issues. All of these things are worries that caretakers shouldn't need to worry about. A special device made and maintained for this purpose that needs a USB port to charge and costs under 50$ is very much worth it.

Why the PineTime?

The PineTime is open source! It's inexpensive! It can run for 2 weeks on a charge! It doesn't have WiFi and so it can go to school! It doesn't beep, so it isn't a distraction to others! It has excellent BT synchronization capabilities, so the watch can be updated reliably when needed!

In short, the PineTime watch with InfiniTime firmware is amazing.

Why a custom server?

Single binary deployment. Can live in a docker container, or just where ever you want it. No complex module dependencies or conflicting OS library versions like NPM or Python, no venv needed. No hair pulling when there are OS or package updates breaking things.

Why ESPHome?

From my personal experience, using ESPHome with esp-idf results in quicker prototyping and easier maintaining of devices out of the gate. Arduino just doesn't offer the groundwork that ESPHome does out of the gate. Template based development also reduces the cost of customization. ESPHome's HTTP interface directly offers debugging and visibility into the device health. All these things make it easy to deploy and maintain.

Components

Go API Server

SQLite-backed REST API and web UI for managing schedules, users, firmware, and bridge status.

ESP32 Bridge

ESPHome firmware on a Waveshare ESP32-S3-Touch-AMOLED-1.8. Fetches reminders over WiFi, syncs to the watch over BLE, with AMOLED display and speaker for event notifications.

Custom InfiniTime Firmware

Modified InfiniTime 1.16 with a reminder service. Stores and fires reminders independently on the PineTime watch.

Features

  • Daily, weekly, and one-time recurrence reminders firing on the watch
  • Priority levels with quiet hours override
  • Watch OTA firmware updates via BLE DFU
  • Watch monitoring - battery level, steps, firmware version
  • Single UX for administrating reminder schedules
